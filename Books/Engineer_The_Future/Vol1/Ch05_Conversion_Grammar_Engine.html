The Conversion Grammar Engine

Chapter V — Wherein the Engine Was Actually Run

Of every mechanism in this project, one alone has actually been run rather than only described. This chapter documents that engine — a grammar for how a thing compresses through ordered layers and returns changed — and the real receipts its execution produced.

I. The Grammar, Bronze-Rated

The six-layer path, actually simulated this project.

24 -> 12 -> 6 -> 3 -> 1 -> 24 Each transition through gate: 1(0)1 Single-crossing rule: only one entity may occupy the active gate.

Bronze, in this project's own lifecycle, does not mean externally proven. It means structurally stable and executable as a rule — and this grammar has now actually been executed, not merely claimed executable.

Real Receipts, Not a Description of What Receipts Would Look Like
Three test entities — eel, frog, fish — were run through the full path this project and every one returned status crossed_changed with a complete five-entry log. A fourth test, not previously run, deliberately attempted a second entity at an already-locked gate: it correctly failed with gate_locked. The single-crossing rule held under an actual attempt to violate it, not just in description.

II. What This Does Not Prove

The engine executing cleanly proves the engine executes cleanly. It says nothing about eels, frogs, positron behavior, or consciousness — each of which remains, as it always was, a candidate application waiting on its own evidence.
