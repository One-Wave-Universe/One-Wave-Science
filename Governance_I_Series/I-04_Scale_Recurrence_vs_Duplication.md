## Purpose

When two nodes appear to describe the same mechanism, that similarity must not automatically be treated as redundant duplication. A repeated structure may indicate: an accidental double-write, the same general rule instantiated at a different scale, genuine cross-scale recurrence, architectural homology across different substrates, or true mathematical scale invariance. These require different repository actions. The correct first step is never immediate merge or immediate dismissal — it is classification.

## The Critical Distinction

Same words ≠ same structure ≠ same mechanism ≠ same mathematics. Shared vocabulary alone proves nothing. Ordered functional recurrence across scales is a real signal worth preserving. A survived rescaling with verified predictions is the only basis for calling something scale-invariant. Do not move directly from shared wording to scale invariance — that turns a genuine recursive hypothesis into numerology with folder names.

## Updated Disposition Taxonomy (supersedes I-03 §Disposition Taxonomy)

1. **Duplicate** — same claim, same scale, same function. No new node; merge/cross-reference.
2. **Renamed Duplicate** — same claim and structure under different terminology at the same scale.
3. **Scale-Specific Instance** — a general rule instantiated at a particular scale, with its own local variables/substrate/failure modes. Preserve both parent and instance.
4. **Scale Recurrence** — the same ordered functional architecture appears across scales, roles map consistently, but mechanism/mathematics not yet shown identical. Preserve all instances, build a cross-scale comparison table, do not claim physical identity.
5. **Architectural Homology** — same functional organization, different substrate/mechanism (e.g., cell membrane vs. personal boundary vs. file permissions). Preserve the comparison, label it explicitly, prohibit claims that one proves the other.
6. **Candidate Scale Invariance** — a cross-scale mathematical mapping is proposed (variables correspond, a rescaling is defined) but the governing form surviving that rescaling is not yet demonstrated.
7. **Supported Scale Invariance** — the scaling transformation preserves the governing relationship and produces verified predictions at more than one scale.
8. **Analogy Only** — comparison is suggestive, no functional or mathematical correspondence shown.
9. **Quarantined Lineage** — reconstructs previously rejected/ungrounded architecture, regardless of terminology.
10. **Unproven Delta** — a genuinely new claim exists but lacks derivation, measurement, or evidence.

A node may carry a primary and a secondary tag (e.g., primary Scale-Specific Instance, secondary Unproven Delta).

## Evidence Ladder for Scale Claims

1. Repeated vocabulary → no scale claim justified.
2. Visual/narrative similarity → analogy only.
3. Functional role mapping → architectural homology.
4. Ordered process recurrence → scale recurrence / self-similarity.
5. Variable correspondence → candidate cross-scale model.
6. Mathematical form preserved under rescaling → candidate scale invariance.
7. Predictions verified at multiple scales → supported scale invariance.

## Required Procedure Before Assigning Disposition

For any apparent duplicate: identify the scale/system/boundary of each node; extract the smallest shared functional pattern as a candidate invariant (not proof); record what changes across scales in a variable-correspondence table; then run the exact-duplication, scale-instance, scale-recurrence, architectural-homology, and scale-invariance tests in that order before assigning a final disposition.

## Final Principle

Repetition at the same scale is usually duplication. Repetition across scales may be recurrence. Recurrence with mapped variables may become a cross-scale model. A cross-scale model whose governing mathematics survives rescaling may qualify as scale invariance. The repository preserves that entire distinction rather than collapsing to either extreme (keep-everything or merge-everything).
