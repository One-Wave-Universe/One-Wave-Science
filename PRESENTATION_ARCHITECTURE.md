# ONE-WAVE PRESENTATION ARCHITECTURE

The repository has one canonical knowledge base and several presentation views. A presentation view never replaces the canonical node or its evidence status.

## 1. Appendices A-G: Wiki Technical Reference

- Canonical source: `Nodes/`
- Human reference: `Wiki_Pages/`
- Machine pack: `AI_Readable_Packs/Appendix_A.md` through `Appendix_G.md`
- Purpose: exact definitions, equations, dependencies, audits, and promotion requirements.

## 2. Books 1-5: Micro to Macro Textbooks

The five scale books use the shared textbook spine:

`Gray reference -> 2D interpretation -> 3D interpretation -> Mathematics -> Predictions -> Yellow Audit -> Future Work`

The PDFs are generated from the current canonical Markdown chapters. They are not independent sources.

## 3. Android Body & Brain: Anime/Manga Technical Build Manual

This is a Proposed Build under I-05. The visual manual may use manga panels, callouts, and character-centered diagrams, but every mechanism must point back to a real node or be labeled as a proposed engineering choice. Building a function does not prove consciousness.

## 4. Engineer the Future: Steampunk Field Manuals

These are build and experiment documents. They emphasize apparatus, specifications, test procedures, and failure conditions.

## 5. One-Wave Times: Investigative Paper Articles

This layer carries Modern-Day Epicycle and Fake-Mustache audits. Each article must separate:

- measured result;
- standard interpretation;
- One-Wave reinterpretation;
- derivation still owed;
- result that would falsify the One-Wave version.

## 6. Proposed One-Wave Consciousness

This remains an Active Hypothesis book. It is not quarantined merely because it is unknown, and it is not promoted merely because the architecture is coherent.

## 7. Musical Universe

The content remains active in `Musical_Universe/`, but its final public presentation format is deliberately undecided. No visual identity should be imposed until the format follows the material rather than decorating it prematurely.

## Governing Rule

The Master List is a navigation summary only. Wiki pages, textbooks, visual build books, newspaper articles, and AI packs all point to the same canonical nodes and proof statuses.

## Terminology legend

`ONE_WAVE_TERMINOLOGY_LEGEND.md` is the canonical Standard-to-One-Wave naming map. Wiki and textbook pages should link to it at first use of a renamed mechanism.
