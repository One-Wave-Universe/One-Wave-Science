Definition

A Break Condition event that preserves a future access pathway rather than producing complete separation. Not every Break produces a Loop Break.

Mathematics

n_LB = number of Loop Breaks n_AL = n_LB (one Access Line per Loop Break) At n_LB=7: Hyperloop entry condition met
Loop counter incrementing with each Return or Loop Break event, capped at n=7 where Hyperloop entry fires.

Math Trail (dependency chain)

The branch point where B-209's Break Condition either dead-ends (Separation) or produces exactly the resource B-213 Access Line depends on — this is the node that decides which.

Recursive Reframing

Direct 1:1 recursive counting relationship already (n_AL=n_LB) — each event increments both counters identically. About as simple a recursive rule as exists in this appendix.

Yellow Audit — what's still open

Conditions distinguishing Loop Break (preserves access) from complete Separation are not formalized. Whether Loop Break quality degrades with each successive break is unresolved. Maximum useful number of Loop Breaks not established.
