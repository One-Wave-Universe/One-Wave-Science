Definition

How a state changes — the three fundamental transitions available to any state, as a signed vector: UP (expansion/Expression), STAY (maintenance/Hold), DOWN (compression/Integration).

Mathematics

Move ∈ {+1, 0, −1} = {UP, STAY, DOWN} UP's Field Action = Expression (B-203) DOWN's Field Action = Compression (B-204) STAY has no dedicated field-action node yet
The three-way vector: UP toward Expression, STAY at the current state, DOWN toward Compression.

Math Trail (dependency chain)

Corrects an earlier version of this same node, which wrongly defined the three moves as {Expression, Compression, Mirror} — Mirror (B-205) is a separate flip mechanism, not one of the three moves, and that error is recorded here rather than silently fixed.

Recursive Reframing

The vector itself ({+,0,−}) is a natural per-step choice at each recursion cycle, feeding directly into B-224's bias mechanism below — but the STAY case in particular still lacks its own dedicated recursive update, unlike UP/DOWN which inherit B-203/204's math.

Yellow Audit — what's still open

No dedicated node exists yet for STAY/Hold as a standalone primitive (it currently only appears inside B-221's six-step cycle). Whether Move and Choice (B-224) are genuinely two separate layers or one layer counted twice is unresolved. The closure claim ('exactly three moves, no fourth') is asserted, not derived.
